Memahami Ukuran Penyimpanan untuk Tipe Data TEXT MySQL

Panduan Lengkap untuk Developer Database

Database 15 Januari 2024

TEXTobjek data, sesuai dengan namanya, berguna untuk menyimpan string teks berbentuk panjang dalam database MySQL. Keempat TEXTtipe objek data dibuat untuk menyimpan dan menampilkan sejumlah besar informasi dibandingkan dengan tipe objek data lainnya yang berguna untuk tugas seperti menyortir dan mencari kolom atau menangani opsi berbasis konfigurasi yang lebih kecil untuk proyek yang lebih besar. Objek yang berbeda TEXTmenawarkan rentang ruang penyimpanan dari 1 byte hingga 4 GB dan tidak dirancang untuk menyimpan nilai komputasi. Biasanya ini digunakan untuk menyimpan deskripsi produk untuk situs penjualan, ringkasan properti untuk database properti, dan teks artikel panjang di situs berita. TEXTobjek paling baik digunakan ketika VARCHARobjek data berbasis string lainnya tidak cukup untuk menangani penyimpanan sejumlah informasi yang diinginkan. Namun, tipe terkecil TEXT, TINYTEXT, memiliki panjang karakter yang sama dengan VARCHAR. TEXTobjek membedakan dirinya dari jenis penyimpanan string lainnya dengan menghilangkan persyaratan untuk menentukan panjang penyimpanan, tidak menghapus byte saat dipilih, dan tidak mengisi ruang karakter yang tidak terpakai untuk penyimpanan disk yang efisien. Karena TEXTobjek tidak disimpan dalam memori server, objek memerlukan overhead data untuk pengambilan. Ukuran berikut mengasumsikan database menggunakan pengkodean UTF-8.
